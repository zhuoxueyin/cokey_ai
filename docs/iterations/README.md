# 每日迭代记录

本目录存放按自然日归档的开发迭代摘要，便于回溯「某天改了什么、为什么改」。

## 命名规范

| 规则 | 说明 |
|------|------|
| 文件名 | `YYYY-MM-DD.md`（与 Git 提交日或实际上线日对齐） |
| 语言 | 简体中文 |
| 篇幅 | 一页以内为宜；细节以代码 diff / PR / `agent.md` 为准 |

## 建议结构（可复制到新文件）

```markdown
# YYYY-MM-DD 迭代摘要

## 背景 / 目标
（可选：当天要解决的问题或需求来源）

## 完成项
- ...

## 修复 / 优化
- ...

## 已知遗留
- ...

## 验证
- [ ] 本地冒烟 / 关键路径
```

## 关联文档

- 系统架构与运维：`agent.md`（版本快照见文首 §零）
- 新渠道接入：`ONBOARDING_SOP.md`
- 渠道协议字段：`PROTOCOL_SPEC.md`

## 每日收尾要求（强制）

- 每日开发结束需同步更新：`agent.md` + 当日 `docs/iterations/YYYY-MM-DD.md`
- 规则已固化到项目规则库：`.cursor/rules/daily-doc-updates.mdc`
